# Cloudinary audit

Generated after the production /admin/ freeze was confirmed live. This audit excludes node_modules, dist, .nuxt, .git, and migration deliverables.

- Files with relevant Cloudinary/CMS references: 1262

- Netlify CMS Cloudinary media library config: 1
- media_folder/public_folder references: 0

## Findings

1. Netlify CMS is configured to upload through Cloudinary in `static/admin/config.yml` using `media_library: name: cloudinary`.
2. Current content stores full Cloudinary URLs, not relative paths. Recent painting records created in April 2026 still contain full `https://res.cloudinary.com/dg6smdedp/image/upload/...` values.
3. The frontend depends on Nuxt Image Cloudinary provider configuration in `nuxt.config.js` and many `<nuxt-img>`/`<nuxt-picture>` usages with `provider="cloudinary"`.
4. The site also hardcodes `https://images.bedfordfineartgallery.com/dg6smdedp/image/upload/...` directly in components/pages. This rollback host must remain untouched, so these references should be mapped to `img.bedfordfineartgallery.com` only after the S3 upload is verified.
5. Cloudinary transformations are present mainly as URL path segments such as `f_auto,q_auto,w_123,h_124` and version segments such as `v1775249264`. Most layout sizing is enforced by stored dimensions, HTML width/height attributes, or pre-cropped assets rather than runtime crop transforms.
6. ArtPlacer receives the painting high-res URL from `pages/painting.vue`; migrated URLs must remain public absolute URLs.

## Required Fixes

- Build `migration/cloudinary-to-s3-map.csv` from Cloudinary inventory before rewriting content.
- Upload all mapped assets to `s3://img.bedfordfineartgallery.com` and verify object availability.
- Replace content and hardcoded URLs using the mapping file, including both `res.cloudinary.com` and `images.bedfordfineartgallery.com` source URLs.
- Replace Nuxt Image Cloudinary provider usage with direct image URLs or a small local helper that accepts already-migrated `img.bedfordfineartgallery.com` URLs.
- Keep `images.bedfordfineartgallery.com` unchanged as rollback.
- Do not unfreeze `/admin/` until the future upload workflow no longer points to Cloudinary.
